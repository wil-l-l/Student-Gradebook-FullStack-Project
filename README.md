## Features

TEACHERS: 
- View their courses
- Publish assignments
- Edit student assignment grades in bulk 
- Edit student assignment grades individually

STUDENTS:
- View their course grades
- View their assignment grades
- Switch the course they are viewing via a dropdown

## Development Insights

What challenges did you face, and how did you overcome them?

One challenge I encountered during development was figuring out how to expand my initial frontend. When I first began this project, I initially had just a small mocked backend using the npm module 'json-server.' Since the project did not have its own backend, I felt limited to what features I could implement on the frontend. The addition of a backend to this project allowed me to vastly expand my frontend.

To continue, because I had expanded the frontend, I was exposed to new problems I didn't initially know solutions to. However, throughout it all, by doing research online I was capable of troubleshooting any problems I encountered. 

Can you describe your development process from its initial stages to final deployment?

The first phase of development was creating and styling primary components of the application. Next, I installed the 'json-server' package, created a JSON file with student grade data, and after fetching that data, displayed it the previously made components. After that, the bulk of the development went towards achieving my desired functionality of the grade table based on the editing mode selected. Finally, there was the deployment of the project in which I had to use Render to create a JSON server that would let me make fetch calls to a backend.

Development of this project initially consisted of developing the functionality involved with school documents. Next, I registered and created teachers, students, courses, and other data associated with schools. Once I established the minimum required functionality of the backend, I went to create the required functionality of the frontend. Finally, I added more non-critical features to both the backend and frontend.
